# Modular Framework Application

Учебный проект, демонстрирующий расширяемую архитектуру приложения на основе модулей и контейнера внедрения зависимостей (Dependency Injection).

## Основная идея

Приложение построено по принципу модульной архитектуры. Ядро системы не содержит логики модулей и работает только через общий контракт.

Каждый модуль:
- имеет имя
- объявляет зависимости от других модулей
- регистрирует службы в DI контейнере
- выполняет инициализацию при запуске

## Архитектура

### Ядро приложения

Ядро отвечает за:

- загрузку модулей
- проверку зависимостей
- построение порядка запуска
- управление DI контейнером

Основные компоненты:
Core/
IAppModule
ModuleCatalog
ModuleLoadException


### Модули

Модули реализуют интерфейс `IAppModule`.

В проекте реализованы следующие модули:

- CoreModule — базовая инициализация
- LoggingModule — журналирование
- ValidationModule — проверка данных
- ExportModule — экспорт данных

### Зависимости модулей
Core
↓
Logging
↓
Validation
↓
Export
## Конфигурация

Список модулей задаётся в файле:


appsettings.json


Пример:

```json
{
  "Modules": [
    "Core",
    "Logging",
    "Validation",
    "Export"
  ]
}
```
### Запуск
Открыть проект в Visual Studio
Собрать решение
Запустить приложение (F5)

### Пример вывода
Модуль Core инициализирован
[Логи] DI контейнер внедрил ILogger
[Логи] Модуль логирования запущен
Модуль проверки данных активирован
Модуль экспорта готов

### Проверка ошибок
#### Отсутствующий модуль

Если в конфигурации указан модуль, которого нет:
FakeModule
приложение выдаст:
Модуль 'FakeModule' не найден
Циклическая зависимость

#### Если зависимости образуют цикл:

Core -> Logging
Logging -> Core

приложение завершится с ошибкой:
Обнаружена циклическая зависимость модулей

### Расширение системы

Чтобы добавить новый модуль:
создать класс
реализовать интерфейс IAppModule
добавить модуль в appsettings.json

Ядро приложения при этом изменять не требуется.
